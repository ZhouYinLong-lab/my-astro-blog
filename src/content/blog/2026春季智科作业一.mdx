# C++ 核心机制与面向对象实战 (一)

在 C++ 的进阶学习过程中，跨越语法的表象，深入理解底层的内存管理、编译机制以及面向对象的基础设计模式至关重要。本文将对近期梳理的几个核心知识点进行系统性、深度地总结，并附带详细的实战代码与注释。

---

## 1. 深入剖析 C++ 程序的四大内存分区

<Info>
**关键概念**：一个编译好的 C++ 程序在操作系统中运行时，其占用的内存空间会被严格划分为四个不同职责的区域。理解它们是解决内存泄漏、段错误（Segmentation Fault）等疑难杂症的前提。
</Info>

* **代码区（Code Segment / Text Segment）**：
  * **作用**：存放程序编译后的二进制机器指令，以及少量的只读常量（如字符串字面量 `"Hello"`）。
  * **特性**：由操作系统管理，具有**共享**（多个程序实例可共享同一份代码）和**只读**（防止程序意外修改自身指令）的特性。
* **全局/静态存储区（Data Segment & BSS）**：
  * **作用**：存放全局变量和被 `static` 修饰的静态变量。
  * **细分**：已经初始化的变量存放在 Data 段；未初始化（或初始化为0）的变量存放在 BSS 段（Block Started by Symbol）。
  * **生命周期**：贯穿整个程序运行期，直到程序结束才由系统回收。
* **栈区（Stack）**：
  * **作用**：存放函数的形参、局部变量以及函数调用的上下文（返回地址等）。
  * **特性**：由编译器**自动分配和释放**，遵循后进先出（LIFO）原则。栈空间通常较小（如 Linux 默认 8MB），极易因极深的递归或过大的局部数组导致“栈溢出（Stack Overflow）”。
* **堆区（Heap）**：
  * **作用**：用于程序运行时的动态内存分配。
  * **特性**：空间极大，但必须由程序员通过 `new`/`malloc` **手动申请**，并通过 `delete`/`free` **手动释放**。若忘记释放，就会造成臭名昭著的**内存泄漏（Memory Leak）**。

## 2. 预处理宏定义的陷阱

<Error>
**注意优先级**：宏定义在 C++ 编译周期的“预处理阶段”仅仅执行**无脑的文本替换**。如果宏定义内部或外层缺少括号保护，在复杂的混合运算中会引发极其隐蔽的逻辑崩溃。
</Error>

来看一个经典的错误示范，假设我们将宏定义最外层的括号去掉：

```diff
- #define max_val(x,y) ((x)>(y))?(x):(y)
+ #define max_val(x,y) (((x)>(y))?(x):(y))
```

**场景重现与灾难推演**：
假设我们在业务代码中执行这样的乘法混合运算：`int result = 10 * max_val(2, 5);`

1. 经过预处理器替换后，代码变为：`int result = 10 * ((2)>(5)) ? (2) : (5);`
2. 根据 C++ 的运算符优先级，乘法 `*` 的优先级高于三目条件运算符 `?:`。
3. 实际的计算顺序变成了：`(10 * ((2)>(5))) ? (2) : (5)`。
4. 因为 `2 > 5` 为假（0），所以 `10 * 0 = 0`。条件判定为假，最终 `result` 被错误地赋值为 `5`，而我们预期的结果应该是 `10 * 5 = 50`！

因此，宏定义的每一个参数，以及整个表达式的外部，**都必须被括号严密包裹**。

## 3. 常量定义：为何偏爱 const

在现代 C++ 规范（如 C++11 及以后）中，强烈建议使用 `const` 甚至 `constexpr` 来替代 `#define` 定义常量。

<Success>
`const` 相比 `#define` 具备三大压倒性的核心优势：

1. **严格的类型检查**：`const` 常量具有明确的数据类型（如 `const int` 或 `const double`），编译器在编译阶段会进行严格的类型匹配检查。而 `#define` 只是字符串替换，极易引发类型错乱。
2. **作用域控制**：`const` 常量遵循 C++ 标准的作用域规则（可被封装在特定的函数、类或命名空间中），有效防止全局命名空间的污染。
3. **调试友好**：`const` 变量在编译后会进入符号表（Symbol Table），在 GDB 或 VS 调试器中可以直接查看其内存地址和当前值；而宏定义在预处理阶段就被抹除了，调试时面对的只是一堆毫无语义的魔术数字（Magic Numbers）。
</Success>

## 4. 性能优化：inline 内联函数

<Info>
**机制解析**：内联机制（inline）是通过将函数体内的汇编代码直接“嵌入”到每一个调用该函数的位置，从而消除传统函数调用带来的昂贵开销。
</Info>

* **消除的开销包括**：参数压栈、寄存器保存、指令指针（PC）跳转、返回地址压栈、以及执行完毕后的现场恢复等底层操作。
* **适用场景**：适用于**体积极小（通常在 10 行代码以内）且被极其频繁调用**的函数（如类的 Getter/Setter 方法）。
* **注意事项**：`inline` 关键字对编译器而言仅仅是一个**建议（Hint）**，而非强制命令。如果函数体内部包含复杂的 `while/for` 循环、`switch` 分支，或者是递归调用，编译器会自动忽略 `inline` 请求，将其作为普通函数编译。

## 5. 动态内存管理：new/delete vs malloc/free

虽然两者都可以申请堆区内存，但在 C++ 面向对象编程中，它们有着不可逾越的鸿沟。

| 特性维度 | `new` / `delete` | `malloc` / `free` |
| :--- | :--- | :--- |
| **语言归属** | C++ 语言的核心内置**运算符（Operator）** | C/C++ 标准库提供的**库函数（Function）** |
| **对象生命周期** | 分配内存后**自动调用对象的构造函数**；<br/>释放内存前**自动调用对象的析构函数**。 | 仅进行冷冰冰的内存划拨和回收，**绝不会触发任何构造与析构逻辑**。 |
| **类型安全性** | 返回对应数据类型的确切指针，绝对类型安全。 | 统一返回 `void*` 万能指针，使用时必须由程序员手动进行强制类型转换。 |
| **空间容量计算** | 编译器自动根据对象类型计算所需的字节数。 | 必须由程序员手动通过 `sizeof()` 计算并传入具体的字节长度。 |

<Warning>
**致命错误**：在 C++ 中，如果对一个包含指针或动态资源的自定义类对象使用 `malloc` 初始化，或者用 `free` 释放，由于没有调用析构函数，会导致该对象内部的子资源彻底成为内存泄漏的死角。
</Warning>

## 6. 面向对象实战：Stack 的两种实现

针对栈（Stack）这一后进先出的经典数据结构，我们分别从**动态数组（物理连续）**与**链表（物理分散）**两个维度进行底层实现，并加入详尽的防御性编程注释。

### 方案一：基于动态数组的扩容栈
利用连续内存空间，支持极快的数组下标访问，通过“容量翻倍”机制来处理满载溢出。这种方式缓存命中率（Cache Locality）极高。

```cpp
#include <iostream>
#include <stdexcept>

class ArrayStack {
private:
    int* data;         // 指向堆区动态数组的指针
    size_t capacity;   // 当前栈的物理最大容量
    size_t topIndex;   // 栈顶指针（即当前栈内实际元素的个数）
    
    // 私有扩容函数：当元素装满时触发，采用均摊复杂度为 O(1) 的倍增策略
    void expand() {
        // 如果当前容量为 0，则初始化为 1；否则容量翻倍
        size_t newCap = (capacity == 0) ? 1 : capacity * 2;
        int* newData = new int[newCap]; // 在堆区开辟全新的、更大的空间
        
        // 将旧数据逐一拷贝至新空间 (现代C++中可使用 std::move 或 std::copy)
        for (size_t i = 0; i < topIndex; ++i) {
            newData[i] = data[i];
        }
        
        delete[] data;    // 极其重要：释放旧的内存区域，防止内存泄漏
        data = newData;   // 指针重定向
        capacity = newCap;// 更新容量记录
    }

public:
    // 构造函数：初始化为空栈
    ArrayStack() : data(nullptr), capacity(0), topIndex(0) {}
    
    // 析构函数：对象被销毁时，自动清理堆区动态分配的数组
    ~ArrayStack() { 
        if(data) delete[] data; 
    }
    
    // 入栈操作
    void push(int x) { 
        if (topIndex == capacity) expand(); // 容量检查
        data[topIndex++] = x;               // 存入数据并后移指针
    }
    
    // 出栈操作
    void pop() { 
        if (topIndex > 0) topIndex--;       // 逻辑删除即可，无需真清空数据
    }
    
    // 获取栈顶元素，附加 const 表示该操作不会修改栈内数据
    int top() const { 
        if (topIndex == 0) throw std::out_of_range("Stack is empty!");
        return data[topIndex - 1]; 
    }
};
```

### 方案二：基于链表的动态栈
通过指针动态连接每一个分散的节点。天然支持无上限增长（仅受限于物理内存），且每一次 `push/pop` 的时间复杂度都是极其稳定的 $O(1)$，没有数组扩容时的性能抖动。

```cpp
#include <iostream>
#include <stdexcept>

// 定义链表的独立节点结构
struct Node {
    int val;        // 数据域
    Node* next;     // 指针域：指向下一个节点
    // 节点构造函数
    Node(int v) : val(v), next(nullptr) {}
};

class LinkedStack {
private:
    Node* head; // 永远指向链表的头部（即栈顶）

public:
    // 构造函数：初始化为空链表
    LinkedStack() : head(nullptr) {}
    
    // 析构函数：利用循环，自顶向下逐个销毁节点，归还内存
    ~LinkedStack() { 
        while(head != nullptr) pop(); 
    }
    
    // 入栈操作：采用链表的“头插法”
    void push(int x) {
        Node* n = new Node(x); // 1. 创建新节点
        n->next = head;        // 2. 将新节点的 next 连上原先的栈顶
        head = n;              // 3. 更新 head 指针，使其重新指向新的栈顶
    }
    
    // 出栈操作：摘除头部节点并释放内存
    void pop() {
        if (!head) return;     // 空栈保护
        Node* t = head;        // 暂存当前栈顶节点
        head = head->next;     // head 指针下移一位
        delete t;              // 彻底销毁原栈顶节点的堆内存
    }
    
    // 获取栈顶数据
    int top() const { 
        if (!head) throw std::out_of_range("Stack is empty!");
        return head->val; 
    }
};
```

## 7. 银行账户类 (Account) 的严谨声明

面向对象编程的核心在于**封装**与**访问控制**。以下是一个典型的 `Account` 类声明，展现了合理的属性私有化与接口公有化。

```cpp
#include <string>

/**
 * @brief 简易银行账户类
 * 展示了 C++ 中基础的面向对象封装特性
 */
class Account {
private:
    // 私有数据成员：外部无法直接通过 对象.balance 篡改数据
    std::string ownerName;  // 储户姓名
    std::string accountNum; // 账户号码，使用 string 防止大数溢出或开头为 0
    double balance;         // 账户核心余额

public:
    /**
     * @brief 构造函数，创建对象时必须提供这三项初始信息
     */
    Account(std::string name, std::string acc, double bal);

    /**
     * @brief 打印账户当前详情。
     * @note 尾部的 const 关键字承诺该函数绝不会修改任何成员变量的值
     */
    void display() const;

    /**
     * @brief 存款操作
     * @param amt 存款金额，内部应有逻辑确保金额大于 0
     */
    void deposit(double amt);

    /**
     * @brief 取款操作
     * @param amt 取款金额
     * @return bool 成功取款返回 true，余额不足返回 false
     */
    bool withdraw(double amt);
};
```
